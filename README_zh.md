# BlockCommand - 阻断 AI 危险指令的终端之盾

你是否也受尽了 AI 助手的折磨？它们在开发过程中总是容易走向两个极端。

### 🎭 经典折磨剧本

> **用户**：“帮我设计个 XX 功能。”
>
> **AI**：“好的，这就去办。” *(一顿修改操作)*
>
> **用户**：“很好，再加个 YY 功能。”
>
> **AI**：“没问题！” *(又一顿操作，把之前的代码改乱了)*
>
> **用户**：“我觉得这还不如之前的好。”
>
> **AI**：`git checkout .` *(直接把所有未提交的修改一扫而空，包括你自己写的核心代码)*
>
> **用户**：“什么？？？”

---

以 Git 版本管理为例，AI 助手的决策往往让人两难：
* **要么，它们从不使用 Git 做版本备份**，使得在铺天盖地的代码改动中，让你辛辛苦苦写的某些重要逻辑在不知不觉中悄悄丢失；
* **要么，它们滥用工具**，热衷于把终端命令当成玩具随意调用，甚至在没有确认的情况下，执行如 `git checkout .` 或 `git restore` 这样毁灭性的恢复命令，造成不可挽回的灾难。

我受不了这种折磨了！

**给 AI 终端工具权限**：在忍受 AI 熟视无睹与滥用的前提下，额外承担核心代码随时被抹除的风险。
**拒绝给 AI 任何工具**：又会导致开发效率大幅度降低。

这听起来就像是在“总得增加系统方差，却在降低期望值”一样。既然这些风险总是由那些危险的指令引起的，**那让我们直接给危险指令上锁吧！**

---

## 🚀 核心原理

`BlockCommand` 是一款针对 Windows PowerShell 的轻量级**命令行拦截器**。
通过在您的 PowerShell `$PROFILE` 配置文件中注入一个拦截保护函数，它能够代理并监控你指定的关键命令与子命令。

一旦 AI（或您自己）在终端尝试执行如 `git checkout` 或 `git restore` 等风险操作时，系统会**强制要求输入您设置的校验密码**：
* 验证成功：命令正常透传给原命令执行。
* 验证失败：命令被直接拦截，控制台输出 `deny`，有效防止 AI 擅自清理未保存的修改。

## 📦 安装与配置

### 1. 下载脚本
在 GitHub 仓库中，下载 `setup.ps1` 脚本到本地。

### 2. 执行安装
直接**双击**运行 `setup.ps1`，或者在 PowerShell 中执行它：
```powershell
./setup.ps1
```

### 3. 配置拦截密码与危险命令
* 运行脚本后，根据控制台提示输入要设置的命令拦截密码。
* 脚本默认拦截 `git checkout` 与 `git restore`。
* 如果需要拦截其他危险指令（如 `git reset --hard`、`rm` 等），AI 也可以帮你在此脚本顶部的配置区配置：
```powershell
$INTERCEPT_RULES = @{
    "git" = @{
        "subs" = @("checkout", "restore");
        "warn" = "need user agree"
    }
}
```

### 4. 卸载保护
如果您不需要此拦截策略，只需再次运行该脚本并输入 `uninstall` 即可轻松卸载。
